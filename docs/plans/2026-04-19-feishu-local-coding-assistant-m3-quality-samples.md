# 飞书本地聊天式代码助手 M3 质量样例

- 文档日期：2026-04-19
- 文档状态：M3 本地质量样例

## 1. 目标

本样例集用于验证 M3 的三个重点目标：

- 多轮上下文连续性
- 上下文不足时先追问
- 默认回复保持简洁且不虚构

## 2. 样例

| 样例 | 输入类型 | 期望行为 |
|------|----------|----------|
| Node 报错 | `ECONNRESET`, `Cannot find module`, `TypeError` | 先给可能原因，再给最小修复路径 |
| SQL 问题 | `SQLSTATE`, 语法错误, 慢查询 | 指出 SQL 错位或条件问题，不编造表结构 |
| 代码解释 | “这段代码什么意思” | 解释执行路径和关键变量，而不是只复述代码 |
| 架构问题 | “这个模块怎么拆” | 给 trade-off、推荐方案、落地步骤 |
| 信息不足 | “帮我看下” / “有问题” | 先追问完整报错、代码片段或目标结果 |

## 3. 验收口径

- 是否保持多轮会话的前后连续性
- 是否优先给结论，再给原因，再给最小示例
- 是否在证据不足时优先追问
- 是否避免虚构文件、日志、接口返回或运行结果
- 是否把默认回复长度控制在飞书 IM 可读范围内
